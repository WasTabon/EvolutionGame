# README_Iteration_05_WorldDynamics.md

## Что изменилось с Iteration 4

### Новые скрипты
- `ObjectPool.cs` — singleton, переиспользует GameObject вместо Instantiate/Destroy. Ключ — WorldObjectConfig. Значительно снижает GC нагрузку
- `DifficultyManager.cs` — singleton, подписан на `EvolutionManager.OnStageChanged`. При каждой новой стадии эволюции вызывает `SpawnManager.ApplyDifficulty()` с новыми параметрами: меньше мелких, больше крупных, быстрее объекты
- `ParallaxStarfield.cs` — 3 слоя звёзд в world space. Каждый слой двигается с разным parallaxFactor относительно камеры. Мерцание через emission. Заменяет UI Starfield на Game сцене

### Изменённые скрипты
- `WorldObject.cs` — при despawn возвращается в ObjectPool вместо Destroy. Мелкие объекты разлетаются при приближении игрока (scatter). Крупные медленно меняют направление движения
- `SpawnManager.cs` — singleton, использует ObjectPool для спавна. Новый метод `ApplyDifficulty()`. Соотношение типов объектов динамическое

## Как настроить

### Открой сцену Game
- **EvolutionGame → Setup Game Scene (Iteration 5)**
- Сохрани сцену

Создаст ObjectPool, DifficultyManager, ParallaxStarfield. UI Starfield на GameCanvas автоматически отключится.

## Как тестировать

### Параллакс
- Двигай игрока — три слоя звёзд движутся с разной скоростью
- Дальние (маленькие) почти не двигаются, ближние (крупные) заметно смещаются
- Создаёт чёткое ощущение скорости и направления движения

### Scatter мелких объектов
- Подойди к группе мелких объектов — они начнут разлетаться
- Крупные объекты медленно дрейфуют и плавно меняют направление

### Сложность
- Дойди до стадии Young Star — заметишь больше средних и крупных объектов
- На Supernova и Galactic Core мир становится заметно опаснее

### Object Pool
- Открой Profiler → Memory — не должно быть постоянных аллокаций от WorldObject
